# 🛠️ Конфигурация NEOVIM для работы с ESP-IDF в Windows

Инструкция по установке всех компонентов для работы с ESP32 в NEOVIM.  
Плагины в Neovim устанавливаются с помощью **Lazy.nvim**.

---

## 🏗️ Установка среды ESP-IDF

Создаем папку `C:\espressif`, в нее скачиваем репозиторий ESP-IDF:

```batch
git clone [https://github.com/espressif/esp-idf](https://github.com/espressif/esp-idf)переходим в папку репозитория:  
```
```  
cd C:\espressif\esp-idf
```  
Выполняем команду установки среды:  
```  
install.bat
```  
Для того что бы развернуть рабочую среду IDF в этом терминале надо выполнить:  
```  
export.bat
```  
Чтобы мы эту команду выполняли где угодно надо добавить ее в системный PATH:  
```  
C:\espressif\esp-idf
```  

**Установка clang от ESP-IDF**  
Инструкция на сайде IDF - 
```
https://docs.espressif.com/projects/esp-idf/en/stable/esp32/api-guides/tools/idf-clang-tidy.html  
```
Перейдите в каталог:  
```  
cd C:\espressif\esp-idf\tools
```  
Выполините команду:
```  
python idf_tools.py install esp-clang
```  
После этого надо запустить export.bat что бы обновить переменные среды  

**Установка NEOVIM**  
Инструкция на сайте - https://github.com/neovim/neovim/blob/master/INSTALL.md  
Установка через chocolatey -  
Устанавливаем с начало chocolatey:  
Инструкция на сайте - https://docs.chocolatey.org/en-us/choco/setup/  
Установите .NET Framework 4.8  
Запускаем PowerShell от имени администратора  
```  
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
```  

После этого можно установить neovim  
```  
choco install neovim
```  
Путь установки надо добавить в PATCH, если его там нет- C:\tools\neovim\nvim-win64\bin  
По этому пути будем запускать nvim.exe  

**Конфигурация neovim**  
Переходим по этому пути:  
```
C:\Users\Admin\AppData\Local\
```
Там создаем папку nvim  
В результате получаем путь - C:\Users\Admin\AppData\Local\nvim  
в эту папку нам надо скопировать содержимое репозитория. В итоге получаем содержимое папки:  

📂 nvim  
 ├── 📁 autoload/                      # Автозагрузочные скрипты  
 ├── 📁 lazy/                          # Установленные плагины  
 ├── 📁 lua/                           # Конфигурация плагинов и модули  
 │    └── 📁 plugins/                  # Файлы конфигурации (Trouble, LSP и др.)  
 ├── 📄 init.lua                       # Главный файл настроек (точка входа) 
 ├── 📄 im-select
 │    └── im-select.exe                # Переключатель раскладки клавиатуры 
 └── 📄 reg-windows-menu               # Скрипты добавления запуска nvim в меню правой кнопки мыши
      ├── neovim_shell_integration.reg # Добавляет позиции в меню
      └── remove_neovim_menu.reg       # Удаляет позиции

**Подключаем утилиту переключения раскладки**
Теперь нам нeужен im-select/im-select.exe, добавляем его в PATH.  
На всякий случай его можно скачать тут:    
```
https://github.com/daipeihust/im-select?tab=readme-ov-file
```
Для проверки в консоли вводим im-select.exe и получаем 1033, значит все ОК.  
Это дает нам автоматическое переключение раскладки на англ. при входе в режим команд.  

**Добавляем nvim в меню при нажатии правой кнопки мыши**
Для этого надо открыть в редакторе neovim_shell_integration.reg  
и убедиться что путь к иконке nvim совпадает с реальной:  
```
"Icon"="C:\\tools\\neovim\\nvim-win64\\share\\nvim\\runtime\\neovim.ico"
```
После чего запускаем его и вносим изменения в реестр windows.

На данном этапе установка и настройка самого VIM закончена.  
Это конфигурация сделанна так что бы подхватывать локальные файлы .clangd .nvim.lua  

.nvim.lua  
├── скрипт запуска терминала с экспортом окружения ESP32  
└── указываем путь к clangd.exe от ESP32  

.clang  
└── указываем пути к include библиотек ESP32  

По итогу получаем гибкую систему - с помощью этих локальных файлов можно  
настроить среду под любой проект, например STM32, C++, QT.. и т.д.  
У каждого проекта будет своя спецефичная конфигурация проекта для VIM.  

# Как работать с этой средой  

## 💻 ТЕРМИНАЛ ESP-IDF
Этот сценарий прописан в локальном .nvim.lua который располагаеться
в корне проекта ESP32. Он открывает терминал и запускает экспорт
среды idf.py.

| Команда | Описание |
| :--- | :--- |
| `Space` + `t` + `e` | **Открыть** терминал ESP-IDF |
| `Space` + `t` + `h` | **Свернуть** терминал (Hide) |
| `Space` + `t` + `s` | **Развернуть** терминал (Show) |

## ⌨️ ВСТРОЕННЫЙ ТЕРМИНАЛ
| Команда | Описание |
| :--- | :--- |
| `:split \| term` | Открыть терминал в горизонтальном сплите (внизу)  |
| `Ctrl + \` + `Ctrl + n` | Выход из режима вставки терминала в Normal mode  |
| `i` | Вернуться в режим ввода команд  |

## 📍 Метки (Marks)
Используются для быстрого перемещения по файлам и проектам.

#### Локальные и Глобальные
| Команда | Описание |
| :--- | :--- |
| `m{a-z}` | Поставить **локальную** метку (внутри файла) |
| `` `{a-z} `` | Прыгнуть к локальной метке |
| `m{A-Z}` | Поставить **глобальную** метку (доступна везде) |
| `` `{A-Z} `` | Прыгнуть к глобальной метке |
| `` `` `` | Прыгнуть к предпоследнему положению курсора |

#### Автоматические и Специальные
> [!TIP]
> Цифровые метки (`0-9`) сохраняются автоматически при выходе из Neovim.
 Каждый следующий выход из файла сохраняется на позиции “0” (название файла и позиция курсора).
При новом выходе локация “0” перезапишется на локацию “1”. Таким образом у нас сохраняется 9 последних локаций,
и мы можем прыгать к ним с помощью `<num>

* **Последние файлы:** Каждая метка от `` `0 `` до `` `9 `` — это позиция в одном из 10 последних закрытых файлов.


* **Список всех меток:** Введите команду `:marks`, чтобы увидеть текущую таблицу переходов.

## 📋 РЕГИСТРЫ И ВСТАВКА  
Использование `Ctrl + r` в режиме вставки или командной строке позволяет быстро вставлять содержимое регистров.  

| Команда | Описание |
| :--- | :--- |
| `Ctrl + r` потом `+` | Вставить из системного буфера обмена (Windows) |
| `Ctrl + r` потом `*` | Вставить из буфера выделения (Select) |
| `Ctrl + r` потом `"` | Вставить последний скопированный/удаленный текст (Безымянный регистр) |
| `Ctrl + r` потом `0` | Вставить текст из последнего "янка" (только `y`, без `d`) |
| `Ctrl + r` потом `w` | Вставить **слово** под курсором |
| `Ctrl + r` потом `a` | Вставить **целое слово** (WORD) со спецсимволами |
| `Ctrl + r` потом `%` | Вставить имя текущего файла (например, `main.c`) |
| `Ctrl + r` потом `/` | Вставить последний текст, который ты искал через `/` |
| `Ctrl + r` потом `=` | Вызов встроенного калькулятора (введи `2+2` и жми Enter) |
